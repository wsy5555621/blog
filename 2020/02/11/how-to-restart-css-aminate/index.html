实际操作中我们会发现，如果没有void element.offsetWidth;这段代码，这个方法是不生效的。可明明改变元素的class也是一种引发reflow的方法啊？我们借助Chrome，可以发现在删除class和增加class时，浏览器会设置一个重新计算样式的任务，然后继续执行代码；在遇上读取元素尺寸时，则直接执行了一次reflow。

class改变不是立即执行reflow的相关操作，这应该算是一个性能的优化，以防一个task中存在太多这样的操作，牺牲流畅度。当我们仅删除再增加class时，浏览器真正执行reflow的时候，发现什么都没有改变，甚至会直接跳过reflow这个步骤。

什么会引起回流

抄一段会引发reflow的操作：

  • insert, remove or update an element in the DOM
  • modify content on the page, e.g. the text in an input box
  • move a DOM element
  • animate a DOM element
  • take measurements of an element such as offsetHeight or getComputedStyle
  • change a CSS style
  • change the className of an element
  • add or remove a stylesheet
  • resize the window
  • scroll
  • changing the font
  • activation of CSS pseudo classes such as :hover (in IE the activation of the pseudo class of a sibling)
  • setting a property of the style attribute
